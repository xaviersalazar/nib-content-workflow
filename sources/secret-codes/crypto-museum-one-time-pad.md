| ![](https://www.cryptomuseum.com/img/blank.gif) |

|     |
| --- |
| **The unbreakable code**<br>The **One-Time Pad**, abbreviated **OTP**, is an [encryption technique](https://www.cryptomuseum.com/crypto/index.htm)<br>in which each<br>character of the plaintext is combined with a character from a random _key_<br>_stream_. Originally described in 1882 by banker Frank Miller ([USA](https://www.cryptomuseum.com/crypto/usa)), it<br>was re-invented in 1917 by [Gilbert Vernam](https://www.cryptomuseum.com/crypto/vernam.htm) and Joseph Mauborgne.<br>The OTP is named after the sheets of paper (pads) on which the key stream is<br>usually printed. It can only be used once. When applied correctly, the OTP<br>provides a truely unbreakable cipher. An automated form, used in combination<br>with a [teleprinter (telex)](https://www.cryptomuseum.com/telex),<br>is known as [_One Time Tape_ (OTT)](https://www.cryptomuseum.com/crypto/ott.htm). |

|     |     |     |
| --- | --- | --- |
| The image on the right shows a page from a ty­pi­cal OTP booklet as it was<br>used during the [Cold War](https://www.cryptomuseum.com/crypto/glossary.htm#cold_war)<br>by [Eastern-Bloc](https://www.cryptomuseum.com/crypto/glossary.htm#eb) agents and spies,<br>in par­ti­cu­lar by agents<br>from the [Soviet Union (USSR)](https://www.cryptomuseum.com/crypto/ussr) and [East-Germany (DDR)](https://www.cryptomuseum.com/crypto/ddr).<br>The booklet consists of a stack of very thin sequentially numbered pages,<br>each of which contains a series of random 5-di­git numbers.<br>Each page could only be used once and had to be destroyed immediately<br>after use.<br>OTPs of [this type](https://www.cryptomuseum.com/crypto/otp/ddr) were often used in combination with<br>Eastern-Block spy radio sets, such as the [DDR Type 2](https://www.cryptomuseum.com/spy/ddr2)<br>and the [Russian R-353 (PROTON)](https://www.cryptomuseum.com/spy/r353). |  | |     |

| **Theory** |
| ![](https://www.cryptomuseum.com/img/blank.gif) |
| ![](https://www.cryptomuseum.com/img/blank.gif) |

|     |
| --- |
| With a one-time pad (OTP), the encryption key has at least the same<br>length as the actual message (i.e. the plaintext) and consists<br>of truely random numbers or letters.<br>Each letter of the plaintext is 'added' to one element from the OTP<br>using modulo-addition.<br>When the key is unknown, this results in a ciphertext that has no relation<br>with the plaintext. At the receiving end, the same OTP is used to retrieve the<br>original plaintext. For this to work, the following rules are mandatory: |

|     |
| --- |
| 1. The OTP should consist of truely random characters (noise). <br>2. The OTP (i.e. the key) must have at least the same length as the plaintext. <br>3. Only two copies of the OTP exist. <br>4. The OTP can be used only once. <br>5. Both copies of the OTP are destroyed immediately after use. |

|     |
| --- |
| When the above rules are strictly obeyed, the OTP is absolutely safe.<br>Combining numbers with the plaintext manually, is a time-consuming task.<br>It is therefore sometimes thought that OTPs are no longer practical.<br>With modern computer technology however, the entire task of enciphering<br>and deciphering can easily be automated, just like it<br>was done in the past on [teleprinter systems](https://www.cryptomuseum.com/crypto/ott.htm).<br>Although it may sound strange, manual OTP ciphers are still being used<br>today (2015) for sending secret messages to agents (spies) via the<br>[Numbers Stations](https://www.cryptomuseum.com/spy/owvl/index.htm),<br>or [One-Way Voice Links (OWVL)](https://www.cryptomuseum.com/spy/owvl/index.htm),<br>that you may have heard on the short-wave radio bands.<br>For a detailed description of the One-Time Pad Cipher and its history,<br>complete with numerous examples, we would like to recommend the<br>excellent paper<br>_Secure Communications with the One Time Pad Cipher_,<br>by Dirk Rijmenants [\[3\]](https://www.cryptomuseum.com/crypto/otp/index.htm#ref_3).<br>➤ [Read now (off-site)](https://ciphermachinesandcryptology.com/papers/one_time_pad.pdf) |

| The major disadvantage of the OTP, is the lo­gis­ti­cal problem of its<br>dis­tri­bu­tion. A unique pair<br>of OTP booklets must be issued and distributed to each individual agent<br>abroad. As the OTP will be destroyed imme­di­a­te­ly after use, sufficient<br>and timely supply of new OTPs has to be guaranteed.<br>During the [Cold War](https://www.cryptomuseum.com/crypto/glossary.htm#cold_war), OTPs were often smuggled<br>into a country by means of a [concealment](https://www.cryptomuseum.com/covert/conceal), such as<br>the one shown in the image on the right. This regular travel kit was cleverly<br>converted into a concealment device by the<br>[East-German Stasi](https://www.cryptomuseum.com/intel/stasi).<br>➤ [More information](https://www.cryptomuseum.com/covert/conceal/travel) |  | |     |

| OTP systems come in many forms and flavours, but most of them consist<br>of either **numbers** or **letters**. An OTP based on numbers is also<br>known as a One-Time Figure Pad (OTFP). When sending text, each letter<br>of the plaintext has to be converted into a number before applying the OTP. |

|     |     |     |
| --- | --- | --- |
| An OTP based on letters is also known as a One-Time Latter Pad (OTLP).<br>An example of an OTLP is shown in the image on the right.<br>It consists of a stack of ~ 30 pages that are stapled to­ge­ther.<br>The conver (just visible at the left) holds a folded alphabet that is used<br>in the translation process.<br>Some OTPs are so small that they can easily be hidden inside a small object.<br>More examples and detailed photographs below. Some OTPs are so small that<br>they can be fitted inside a slide frame. |  | |     |

|     |
| --- |
| When used properly, OTPs are inherently safe. But when they are used<br>incorrectly, e.g. by reusing them, they can be a cryptologic nightmare.<br>A good example of improper use is described by Matt Blaze in an article<br>about the book _Compromised_ by former [FBI](https://www.cryptomuseum.com/intel/fbi/index.htm) counterintelligence agent<br>Peter Strzok [\[4\]](https://www.cryptomuseum.com/crypto/otp/index.htm#ref_4). He describes how secret OTP-encrypted messages were sent<br>to Russian illegals 1 in the [US](https://www.cryptomuseum.com/crypto/usa), via a<br>[Numbers Station](https://www.cryptomuseum.com/spy/owvl/index.htm) in Cuba<br>which had a good coverage throughout the [US](https://www.cryptomuseum.com/crypto/usa).<br>The station would broadcast message in [morse code](https://www.cryptomuseum.com/radio/morse/index.htm)<br>and voice 24 hours a day, interleaving real messages with dummy traffic, so<br>that it was not revealed to an interceptor how often and how many real messages<br>were sent. However, the [FBI](https://www.cryptomuseum.com/intel/fbi/index.htm) discovered (as did others) that dummy traffic could<br>easily be discriminated from real traffic, as it did not contain the number 9.<br>This was probably caused by a faillure of the random number generator used<br>for the dummy traffic, or a bug in the software that handled it.<br>With this knowledge, the [FBI](https://www.cryptomuseum.com/intel/fbi/index.htm) was able to correlate the real messages to<br>the times that messages were decoded by a Russian illegal couple they had under<br>surveillance. It eventually led to the arrest of an several Russian illegals in<br>the [US](https://www.cryptomuseum.com/crypto/usa) and Canada. |

| OTP booklets, such as the one shown above,
have been captured during the Cold War by Western intelligence
agencies on a number of occassions. One documented example is the capture
of a Dutch man, who
acted as an East-German agent in The Netherlands, in 1969.
When he was finally exposed, the Dutch intelligence service
[BVD](https://www.cryptomuseum.com/intel/index.htm#bvd)
(now: [AIVD](https://www.cryptomuseum.com/intel/aivd))
found a partly used OTP booklet in his home, along with a fully operational
[R-353 spy radio set](https://www.cryptomuseum.com/spy/r353/index.htm),
a [burst encoder and cassettes](https://www.cryptomuseum.com/spy/r353/burst.htm) [\[5\]](https://www.cryptomuseum.com/crypto/otp/index.htm#ref_5).
